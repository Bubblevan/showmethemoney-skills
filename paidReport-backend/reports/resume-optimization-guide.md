# 简历优化建议报告

**发布日期**：2025年4月  
**版本**：v1.0

---

## 引言

在竞争激烈的就业市场中，一份优秀的简历是获得面试机会的关键。本报告针对技术岗位，提供系统化的简历优化建议。

---

## 1. 简历结构优化

### 黄金布局

```
1. 个人信息（姓名、联系方式、GitHub/博客链接）
2. 求职意向（目标岗位，一句话描述）
3. 核心技能（技术栈，按熟练度排序）
4. 工作经历（STAR 法则描述）
5. 项目经验（量化成果）
6. 教育背景（学历、相关课程）
7. 其他（证书、开源贡献等）
```

---

## 2. 内容优化技巧

### 2.1 量化成果

**避免**：
- 负责后端开发
- 优化了系统性能

**改为**：
- 负责后端开发，服务支撑日活 10万+ 用户
- 优化系统性能，API 响应时间从 200ms 降至 50ms

### 2.2 关键词优化

根据目标岗位 JD，匹配关键词：
- 技术栈：Python、Go、微服务、Kubernetes
- 方法论：敏捷开发、CI/CD、TDD
- 软技能：跨团队协作、技术方案设计

### 2.3 STAR 法则

描述项目时使用：
- **S**ituation：项目背景
- **T**ask：你的任务
- **A**ction：采取的行动
- **R**esult：取得的结果

---

## 3. 常见错误

### 错误清单

- [ ] 简历超过 2 页
- [ ] 使用过于花哨的模板
- [ ] 堆砌技术名词但没有实际应用
- [ ] 没有针对岗位定制
- [ ] 联系方式错误或不易联系
- [ ] 拼写和语法错误

---

## 4. 技术岗位特殊建议

### 4.1 程序员

- 突出代码质量和工程能力
- 展示技术博客或开源贡献
- 链接 GitHub 展示项目

### 4.2 产品经理

- 强调数据驱动决策能力
- 展示产品设计案例
- 体现跨部门协调能力

### 4.3 设计师

- 附上作品集链接
- 展示设计思考和过程
- 突出用户体验成果

---

## 5. ATS 系统优化

### 什么是 ATS

 applicant tracking system（申请追踪系统），自动筛选简历的工具。

### 优化策略

1. 使用标准字体和格式
2. 避免图表、表格、图片
3. 确保关键词与 JD 匹配
4. 使用标准章节标题
